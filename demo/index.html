bundle-cost-cli

Print the real over-the-wire cost of your build —
and the delta on every save.

A tiny CLI that prints raw, gzip, and brotli sizes for any built file — and, in watch mode, a colored Δ on every save. No bundler plugin, no config, no telemetry.

Why

You add one date library, hit save, and your bundle quietly grows 60 KB. You find out weeks later. bundle-cost tells you in the moment.

Real bytes, not estimates

Reads each file as-is and runs it through Node's built-in zlib. Gzip + brotli are what your CDN serves; that's what gets measured.

Watch & diff

The first render is your baseline. Every save adds a Δ column — green when you shrank it, red when you grew it.

Budget gate

Pass --limit 50KB to fail with exit 1 when the gzipped total blows your size budget. CI-friendly.

Install

$ pnpm add -g bundle-cost-cli
$ pnpm dlx bundle-cost-cli dist/*.js  # or run ad-hoc without installing

Requires Node 18+. Works on macOS, Linux, and Windows.

Common uses

# after a build, see the breakdown
$ bundle-cost dist/*.js

# keep it open in a terminal while you work
$ bundle-cost dist/*.js --watch

# fail CI when you blow your size budget
$ bundle-cost dist/index.js --limit 50KB

# include brotli alongside gzip
$ bundle-cost dist/*.js --brotli
